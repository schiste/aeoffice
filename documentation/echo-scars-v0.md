# Echo Scars (v0 Catalog Draft)

Echo Scars are persistent (`untuned`) Hero-side consequences triggered by Viral Load forced-return events.

Design goals (v0):
- Always **bonus + malus** (bonus may be “silly”, but mechanically real).
- Can affect the Hero’s exploration/combat and indirectly affect Base output through the Hero (e.g., Hero band multipliers).
- Scars can stack (`stacks` per scar ID).
- Removal is possible in v0 but is **consumable-only**; exact removal mechanics are designed later.

Selection (v0):
- Eligibility can depend on conditions (TBD).
- When a scar is awarded, the game can present **3 candidates** and the player chooses one (final pipeline is defined in `documentation/object-templates.json`).

This document is descriptive; the structured template lives in `documentation/object-templates.json` under `EchoScarSystemConfig`.
