# Object Types & Taxonomy (Draft)

This document creates a shared vocabulary for “what exists” in AD&D: simulation objects, content objects, resources, and their persistence rules.

Scope:
- This is a **design taxonomy**. It should stay stable even as implementation details change.
- “Persistence” refers to behavior on **Tuning** and across **save slots**.

---

## 1) Persistence Tags (Tuning Affinity)

From `mechanics/specifications.md`:
- `tuned`: run-scoped; reset/destroyed on Tuning (e.g., crew, stored band pools, safe spots, active expeditions).
- `detuned`: persists as a shell but becomes inert; must be re-attuned to function (e.g., many Base stations).
- `untuned`: persists unchanged (e.g., Hero, map knowledge, some long-term progression).

Important clarifications adopted so far:
- **Band stored pools** reset on Tuning (run-scoped).
- **Materials can define their own affinity** (do not assume all materials reset).
    - Example decision: **Stone is `untuned` (persists)**.
- **Map node stocks are persistent world-state within a save slot** (new save slot starts fresh).

## 2) Resource Types

### 2.1 Band Resources (Numeric Pools)
Run-scoped numeric pools generated by staff via the Crystal:
- `Bassline` (low band) — drives bubble reach via stored pool.
- `Chorus` (mid band) — powers Base systems (starts locked at 0 early).
- `Harmonics` (high band) — efficiency/tuning (starts locked at 0 early).

Notes:
- Early progression: **only Bassline is generated** at first.
- Bands are generated by the **Crystal/Crystal Circle** early; other buildings may modify efficiency later.
- Expeditions return **loot/materials**, not band resources directly (early decision).

### 2.2 Materials (Numeric Pools)
Persistent or run-scoped numeric pools used mainly for construction/progression gating.
- `Stone` — first material; **persists across Tunings**; `Stone_cap_base = 1000`.
    - Cap behavior: collection is blocked at cap; in-progress actions can pause and require player intervention.
    - Harvesting alignment: **Drummer bonus / Vocalist neutral / Synth malus** (per-material mapping).
- `Water` — material pool (units: liters); **persists across Tunings**; v0 cap `5L` (starter).
    - v0 source: Base tile collection (available from run start; tile has stock+regen).

Default rule (decision):
- Materials default to `untuned` persistence (persist across Tunings), but can be overridden per material.

Sources (currently decided at a concept level):
- Base-tile scavenging (manned; not counted for offline catch-up).
- Map harvesting (Hero and/or expeditions); terrain has ambient trickle; nodes are concentrated deposits.
- Expeditions can return materials while offline (subject to caps).

### 2.3 Craft Tiers (Inventory or Pools; TBD)
Crafting tiers are intentionally omitted for now and will be redesigned alongside items/gear.

### 2.4 Meta-Progression (Persistent)
Long-term progression variables referenced in specs:
- `Resonance (R)` — recomputed on Tuning; cross-run multiplier.
- `Fragments (F_unspent)` — persistent; contributes to legacy effects.
- `Refinement (P)` — persistent; affects legacy and gates/unlocks.

## 3) Actor Types

### 3.1 Hero (Untuned)
- Can be assigned to Base roles (e.g., Crystal Circle, Fire Pit) when not exploring.
- Early: generates **Bassline only**; one band at a time.
- Reassignment cooldown: ~3s (draft).
- Hero XP/levels persist across Tunings (benefit: stronger early-run Hero).

### 3.2 Crew (Tuned)
- Crew classes: Drummer / Vocalist / Synth.
- Used for Base staffing and for expeditions.
- Crew dies on Tuning (hard reset).

## 4) Base Objects

### 4.1 Buildings (Physical, Placeable)
Examples currently referenced:
- `The Studio` — permanent starter building; v0 starts damaged with `bunks_capacity = 0` until restored (first restore provides `+15` bunks).
- `Fire Pit` — Vibes building, build cost in Stone, `max_count = 1`.
- `Crystal Circle` — staffing structure for band generation (3 slots at L0).

Bunks v0 note:
- The Hero consumes `1` bunk; recruiting is not hard-limited by bunks, but overcapacity triggers BadVibes.

Cost scaling (shared rule, per building type):
- `Cost = BaseCost * CountMult_building(n_existing) * TuningMult_building(N)`
    - `CountMult_building` default is mild power curve (`p_count_default = 1.2`), overridden per building.
    - `TuningMult_building` is per building type, piecewise grace + sublinear ramp.

### 4.2 Stations (Operational Roles)
“Stations” are interactive operational units that may have staffing slots, upkeep, and outputs.
- Many stations are Chorus-powered later; early exceptions can exist (e.g., Fire Pit has no Chorus upkeep early).

## 5) World/Map Objects

### 5.1 Tiles / Terrain
Terrain types exist and define impedance/occlusion behavior for bubble propagation.

Per-tile exploration state (v0):
- Tiles have persistent investigation/exploration state within a save slot (levels are monotonic).
- Exploration can grant one-time **global unlocks** (e.g., dungeons) that do not reveal tiles by themselves.
- Dungeon availability can still depend on the associated tile being “accessible” (v0: in bubble + Hero visited + access conditions met).

### 5.2 Resource Fields
- Ambient resource presence: low stock, low efficiency harvesting.
- Nodes: high stock and/or higher efficiency; example Stone nodes: Small Quarry, Modern Quarry, Mine, Open Mine.
- Node stocks are persistent (per save slot).
- For resources that regenerate naturally, regeneration advances in real time while offline.
    - Stone nodes do not regenerate by default.

### 5.3 Bubble / Safe Coverage
- Bubble coverage is computed from stored Bassline (automatic early).
- Safe spots and relays exist later (Chorus-powered, tuned objects).

---

## 6) Expedition Objects

### 6.1 Expeditions (Tuned)
- Crew-only, parallel, time-based runs.
- Offline/catch-up applies (subject to caps).
- Expeditions return **loot/materials**, not band resources directly (early).

Loot items (v0 note):
- Some progression actions consume loot items (example: `skin` for “Polish the Crystal Base”).

Open design question:
- Reward/punishment layer beyond brownout/incapacitation (explicitly deferred).
